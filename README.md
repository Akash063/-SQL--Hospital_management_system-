# -SQL--Hospital_management_system-
🏥 Hospital Management System (SQL Project)

This project is a Hospital Management System built using SQL, designed to efficiently manage and analyze hospital data including patients, doctors, departments, and appointments. The system demonstrates how relational databases can be used to organize healthcare operations and generate meaningful insights.

📌 Project Overview

The database simulates real-world hospital workflows by storing structured data across multiple related tables. It enables users to perform operations such as tracking patient records, managing doctor availability, and analyzing hospital performance.

⚙️ Features
Patient record management (admissions, discharge details)
Doctor and department management
Appointment scheduling system
Hospital-wise data tracking
Advanced SQL queries for data analysis
Reporting insights (e.g., top departments, patient count, doctor availability)
🗂️ Database Structure

The project includes multiple relational tables such as:

Patients
Doctors
Hospitals
Departments
Appointments

These tables are connected using primary and foreign keys to maintain data integrity.

📊 Key SQL Operations
Data retrieval using SELECT, JOIN, GROUP BY
Aggregations (COUNT, AVG, SUM)
Subqueries and nested queries
Ranking and filtering (TOP, LIMIT)
Data cleaning and transformation
🚀 Insights Generated
Total number of patients across hospitals
Average number of doctors per hospital
Top-performing departments based on patient count
Doctor workload and availability trends
🛠️ Tools & Technologies
SQL (MySQL / PostgreSQL)
CSV data import
Database normalization techniques
